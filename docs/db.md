# Schemas

## Posts

```text
id                primary key
userId            foreign key
body              text
hasAttachments    boolean
views             non-negative integer
createdAt         datetime
updatedAt         datetime
```

## Attachments

```text
id                primary key
postId            foreign key
type              enum
fileUrl           text
createdAt         datetime
updatedAt         datetime
```

- `type` can be ["image", "gif", "video"]

## Profile

- Works along with `user` table generated by Better Auth

```text
id                  primary key
userId              foreign key
avatarUrl           text
bannerUrl           text
bio                 text
isGlimpseVerified   boolean
createdAt           datetime
updatedAt           datetime
```

- Better Auth must have extra attribute `username` which must be unique. Update can be handled separately or with auth client.

## Comments

```text
postId              foreign key
userId              foreign key
parentCommentId     foreign key
body                text
createdAt           datetime
updatedAt           datetime
```

- Only one level of nesting should be allowed. i.e. if parentCommentId also has a parentCommentId, operation must be abandoned.

## Likes

1. Posts

```text
postId              foreign key
userId              foreign key
createdAt           datetime
```

2. Comments

```text
commentId           foreign key
userId              foreign key
createdAt           datetime
```

- both the foreign keys in a pair form the primary key, therefore are always unique.

## Followers

```text
followerId          foreign key
followingId         foreign key
createdAt           datetime
```

## Bookmarks

```text
postId              foreign key
userId              foreign key
createdAt           datetime
```
